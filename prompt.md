# 《天龙八部》续写项目 Prompt 设计思路
基于多 Agent 协同的通用小说续写框架实现将分为 **“初始化阶段（构建记忆库）”** 和 **“续写阶段（多 Agent 协同生成）”** 两大部分，流程如下：



## 第一阶段：知识库构建（Stage1Prompts）
本阶段 Prompt 核心目标是从《天龙八部》原著中提取结构化知识库，为后续续写提供底层设定支撑，避免创作脱离原著框架。

| Prompt 类别 | 设计核心逻辑 | 关键约束 |
|------------|--------------|----------|
| 文本摘要生成（SUMMARY） | 聚焦核心剧情、关键人物、核心场景，输出精简的结构化摘要，为全局主线梳理打基础 | 100字内核心剧情、JSON 格式输出、无冗余文字 |
| 全局主线/结局生成（GLOBAL_SUMMARY） | 锚定全书主线与结局，明确续写的时间锚点和剧情边界 | 500字内主线、200字内结局、仅 JSON 输出 |
| 人物库提取（EXTRACT_CHARACTERS） | 标准化人物维度（姓名、性格、武功、经历、口头禅），为核心人物人设铁则提供数据支撑 | 至少25个主要人物、JSON 数组格式、维度信息完整 |
| 武功库提取（EXTRACT_KUNGFU） | 标准化武功维度（名称、类型、使用者、描述），避免续写中出现武功错配/战力失衡 | 至少30种武功、50字内描述、关联对应使用者 |
| 世界观提取（EXTRACT_WORLD） | 拆解时代、势力、地理、规则四大维度，固化续写的世界观边界 | 锚定北宋哲宗年间、势力状态清晰、JSON 结构化输出 |
| 文风样本提取（EXTRACT_STYLE） | 提取原著景物、打斗、对话的代表性样本并分析，转化为可执行的文风规则 | 按场景/打斗/对话分类、附带风格分析、JSON 输出 |

## 第二阶段：多Agent协同续写（Stage2Prompts）
本阶段基于第一阶段的知识库，设计多Agent分工的 Prompt 模板，实现“大纲生成→内容创作→合规校验”的全流程创作管控。

### 静态规则层：核心约束固化
将第一阶段提取的知识库转化为“强制约束”，作为所有创作类 Prompt 的底层规则，避免核心设定偏离：
- **黑名单人物（FORBIDDEN_CHARACTERS）**：明确绝对禁止出现的人物（如乔峰、玄慈等已离世/结局定型的角色），规避时间线/剧情冲突；
- **核心人物铁则（DUANYU_RULES/XUZHU_RULES）**：拆解段誉、虚竹的核心人设（武功、性格、行为准则、称呼规范、结局状态），每条均为“绝对禁止/必须遵守”（如段誉仅用六脉神剑+凌波微步、虚竹对段誉仅称“三弟”）；
- **世界观铁则（WORLD_RULES）**：锚定时代背景、逍遥派设定、战力体系（段誉+虚竹=战力天花板），禁止跨金庸作品设定冲突；
- **文风铁则（STYLE_RULES）**：将金庸文风转化为可执行规则（半文半白长句、景物烘托开头、200-300字段落、打斗需含招式/内力细节等）。

### 动态模板层：多Agent分工设计
| Agent 角色 | 设计核心逻辑 | 核心约束/输出要求 |
|------------|--------------|------------------|
| 大纲生成Agent（OUTLINE_AGENT） | 以“起承转合”为核心结构，嵌入静态规则约束，强制原创势力/人物，保证剧情符合天龙内核 | 1. 时间锚定：原著结局后3年；<br>2. 结构要求：「起」铺垫动机、「承」触发核心事件、「转」对峙+打斗+反派身世、「合」解决冲突+留伏笔；<br>3. 输出：JSON 格式大纲+原创元素（势力/人物/核心秘密） |
| 大纲校验Agent（OUTLINE_VERIFY） | 一票否决制，针对大纲的合规性、完整性、细节丰满度逐一校验 | 仅输出 PASS 或具体违规点+修改意见 |
| 正文生成Agent（SECTION_WRITER） | 全新生成指定段落，优先约束字数范围，嵌入静态规则+文风铁则，保证内容贴合结构要求 | 1. 字数强制：严格控制在指定区间；<br>2. 内容：覆盖对应“起承转合”段落核心要求；<br>3. 输出：仅正文，无标题/解释/Markdown |
| 正文扩写Agent（SECTION_EXPANDER） | 不改变原有情节，通过增加景物/心理/打斗细节扩写至指定字数 | 同正文生成Agent，新增“不修改核心情节”约束 |
| 分段校验Agent（SECTION_VERIFY） | 针对单段正文，校验人设/世界观/文风/核心内容覆盖度 | 输出 JSON 格式结果（PASS/FAIL）+ 错误细节 |
| 全文终审Agent（CONTENT_VERIFY） | 全局视角校验全文合规性、叙事结构、总字数、天龙内核 | 输出 PASS+润色建议 或 具体违规点 |
| 回目标题生成（CHAPTER_TITLE） | 复刻《天龙八部》七言对仗的章回标题风格 | 仅输出标题，无冗余内容 |
